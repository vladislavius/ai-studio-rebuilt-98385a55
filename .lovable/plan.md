

# Анализ заглушек и недоработок в портале

## Академия — заглушки и неработающие элементы

### 1. Ответы студента не сохраняются
Поле «Ваш ответ» (RichTextEditor для типов write/demo/clay_demo) в `CourseStudyView.tsx` — **заглушка**. `onChange={() => {}}` — текст никуда не сохраняется, при переключении шага или перезагрузке теряется.

### 2. Тесты (quiz) нельзя создать через интерфейс
В `CourseChecksheet.tsx` (редактор) нет UI для добавления `quizQuestions` к шагу. Поле существует в интерфейсе `ChecksheetItem`, компонент `QuizStep.tsx` работает, но заполнить вопросы теста невозможно — только через AI-генерацию (если генератор их создаёт).

### 3. Блокировка следующих шагов отсутствует
Документация (AcademyDocsPage) описывает «нельзя перейти к следующему шагу без выполнения текущего», но в `CourseStudyView.tsx` студент может кликнуть на любой шаг в боковой панели — блокировки нет.

### 4. Твиннинг (парные упражнения) — полностью заглушка
Описан в документации (модуль 7), но нет ни одного компонента, таблицы или интерфейса для:
- Подбора пар
- Календаря сессий
- Формы обратной связи

### 5. Демо-инструменты (whiteboard, mind map) — заглушка
Документация упоминает интеграцию с whiteboard/mind map/блок-схемами для шагов demo и clay_demo, но фактически студент просто описывает демо текстом.

### 6. Роли пользователей (студент/супервизор/автор/админ) — не разделены
Описаны 4 роли, но реально используется только `isAdmin` (true/false). Нет отдельной роли супервизора — админ видит всё. Студент (не-админ) не видит вкладку «Сотрудники», но видит чужие курсы.

### 7. Привязка студента к супервизору — отсутствует
Чат «с куратором» есть, но нет механизма назначения конкретного куратора/супервизора к студенту или курсу.

### 8. Сертификация — частично работает
Сертификат генерируется (SVG), но поле `certified` в `course_progress` никогда не устанавливается в true через интерфейс — нет кнопки «Сертифицировать» для супервизора.

### 9. Программы (последовательности курсов) — отсутствуют
Документация описывает «Программа → Курс → Модуль → Шаг», но нет таблицы `programs` и нет UI для группировки курсов в программы с prerequisites.

### 10. Версионирование курсов — отсутствует
Упомянуто в документации, не реализовано.

---

## Остальной портал — заглушки

### 11. Сотрудники — работает
Список, добавление, редактирование, кандидаты, онбординг, документы, дни рождения — всё подключено к БД и функционально.

### 12. Оргструктура — работает
Департаменты, вложенность, редактирование, привязка сотрудников — работает.

### 13. Статистики — работает
Полноценный модуль: ввод значений, графики, тренды, состояния, CSV-экспорт/импорт, «боевые планы».

### 14. Админшкала — работает (localStorage)
Данные хранятся в `adminscale_scales` в БД. Шаблоны, редактор, экспорт/импорт JSON — работает.

### 15. Настройки — работает
Терминология, кастомизация лейблов.

## Резюме: что является заглушкой

| Компонент | Статус |
|-----------|--------|
| Ответы студента (write/demo) | Заглушка — не сохраняются |
| Quiz-редактор | Заглушка — нет UI создания |
| Блокировка шагов | Заглушка — нет блокировки |
| Твиннинг | Заглушка — полностью описание |
| Whiteboard/mind map для демо | Заглушка |
| Разделение ролей (4 роли) | Частичная заглушка |
| Назначение супервизора | Заглушка |
| Кнопка «Сертифицировать» | Заглушка |
| Программы курсов | Заглушка |
| Версионирование курсов | Заглушка |
| Всё остальное (сотрудники, оргструктура, статистики, админшкала) | **Работает** |
