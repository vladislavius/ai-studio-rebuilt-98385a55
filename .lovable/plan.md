

# Академия — Контрольные листы по технологии Хаббарда

## Текущее состояние
- Есть таблицы `courses` (с `sections` jsonb) и `course_progress` (`completed_sections` jsonb, `progress_percent`, `certified`)
- Академия показывает список курсов-карточек, можно создать/удалить/опубликовать
- Нет: редактора контрольного листа, прохождения курса студентом, назначения, дашборда прогресса

## Концепция: Контрольный лист (Checksheet)

Каждый курс — это контрольный лист. Состоит из пунктов (items), каждый пункт имеет:
- Номер по порядку
- Тип действия: `read` (прочитать материал), `write` (написать эссе/ответ), `demo` (продемонстрировать), `drill` (практическое упражнение), `starrate` (проверка звёздной оценкой)
- Текст задания (что именно сделать)
- Ссылка на материал (опционально — текст статьи, URL видео)
- Студент двигается последовательно, отмечая каждый пункт как выполненный

Данные хранятся в существующем поле `courses.sections` jsonb — массив объектов-пунктов. Миграции БД не нужны.

## План реализации

### 1. Рефактор Academy.tsx — навигация по вьюхам
Добавить state `view`: `'list' | 'detail' | 'study'` и `selectedCourseId`.
- `list` — текущий список курсов (+ табы "Курсы" / "Мой прогресс")
- `detail` — редактор контрольного листа (админ)
- `study` — прохождение контрольного листа (студент)

### 2. CourseChecksheet.tsx — редактор контрольного листа (админ)
- Заголовок курса, описание, HST-флаг
- Список пунктов контрольного листа с drag-перемещением (или кнопки вверх/вниз)
- Для каждого пункта: порядковый номер, тип (выпадающий: прочитать/написать/демо/упражнение/звёздная оценка), текст задания, материал (textarea для встроенного текста или URL)
- Кнопки: добавить пункт, удалить, сохранить весь лист
- Сохраняется в `courses.sections` jsonb
- Кнопка "Назначить сотрудникам" — мультиселект из `employees`, создаёт записи в `course_progress`

### 3. CourseStudyView.tsx — прохождение контрольного листа (студент)
- Слева: список пунктов с номерами, иконками типа, статусом (выполнен / текущий / не начат)
- Справа: содержание текущего пункта — текст задания, встроенный материал
- Кнопка "Выполнено ✓" — отмечает пункт в `course_progress.completed_sections`, пересчитывает `progress_percent`
- При 100% — автоматически ставит `completed_at`, предлагает сертификацию
- Прогресс-бар сверху

### 4. Таб "Мой прогресс" на странице Академии
- Для админа: таблица всех назначений (сотрудник × курс × % × статус)
- Фильтры по статусу: не начат / в процессе / завершён / сертифицирован

### 5. Интеграция с карточкой сотрудника
- В секции "Карта развития" в EmployeeForm.tsx — показать назначенные курсы с прогресс-барами
